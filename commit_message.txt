fix(app): Revert to absolute imports to fix ImportError

The previous commit introduced relative imports (`.skid_logic`, `.floorboard_logic`) in `app.py`. This caused an `ImportError` when running the Streamlit application directly as a script, as it lacks a known parent package for relative imports.

This commit reverts the import statements in `app.py` back to absolute imports (`skid_logic`, `floorboard_logic`). This allows the application to run correctly when launched with `streamlit run app.py`, assuming the logic files are in the same directory.

Also included minor visualization font color improvements and tolerance additions from the attempted v0.3.18 update.
